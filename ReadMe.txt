3. 数据流转与映射表脚本运行时会自动从“外部数据源”搬运或读取文件到项目目录。数据模块源文件名 (在 Raw Data 中)目标文件夹 (在 1_Data 中)处理逻辑全A股基础Pctchg_连续表.xlsx000_common_data自动复制。若存在且被占用则跳过。Price_连续表.xlsx000_common_data同上。Amount_连续表.xlsx000_common_data同上。市场情绪涨跌停数据.xlsx002_市场情绪跟踪每次运行强制覆盖复制。指数跟踪指数跟踪第五版-提取表.xlsx003_指数量价跟踪每次运行强制覆盖复制。指数历史指数跟踪第五版 - 前期表.xlsx003_指数量价跟踪不复制。直接读取目标文件夹内的现有文件。4. 关键业务逻辑设定 (Hardcoded Logic)这些是写在 build_report.py 和 HTML 中的核心规则：A. 数据清洗规则停牌处理：在读取 Pctchg_连续表 时，程序强制将 -100 的值替换为 0 (视为平盘/停牌)，以免干扰跌幅统计。指数宽表转置：读取指数 Excel 时，会自动识别“指数”列并执行转置（Transpose），将横向排列的日期转换为纵向时间序列。指数历史拼接：自动将“前期表”和“提取表”按时间轴拼接，并去除重复日期（保留最新）。B. 指标计算规则涨跌停家数封顶：超过 300 家时，强制显示为 300（用于图表美观）。成交额变化率：定义为 (本期日均成交额 - 上期同天数日均成交额) / 上期...。市盈率分位数：计算池范围为 [数据起始点 ~ 用户选择的结束日期]。量价背离：对比“成交额前100名涨跌幅中位数”与“成交额后3000名涨跌幅中位数”的差值。C. 前端显示格式成交额单位：全A总成交额：万亿元 (保留2位小数)。个股/指数成交额：亿元 (保留2位小数)。市盈率截断：图表 Y 轴强制截断在 -50 到 300 之间，过滤极端值。颜色规范：🔴 上涨/高估/Top100：#ef4444 (红色)🟢 下跌/低估/Bot3000：#22c55e (绿色，两市分布中用 #86efac 浅绿)🔵 最新值/中性：#3b82f6 (蓝色)⚪ 辅助/背景：#e2e8f0 (浅灰)5. 指数分组定义 (预设)在 build_report.py 中预定义了以下分组：核心宽基指数 (上证、沪深300等 5个)扩展的宽基指数 (含科创50等 10个)全部宽基指数 (含北证50、微盘股等 18个)主要行业指数 (信息、医药等 10个一级行业)全部行业指数1 (包含细分行业)全部行业指数2 (包含概念板块如昨日涨停等)6. 维护建议添加新指数：如果在 Excel 中添加了新指数，需同步在 build_report.py 的 user_defined_groups 字典中添加对应的简称，程序会自动进行模糊匹配。更新历史数据：如果 指数跟踪第五版 - 前期表.xlsx 有变动，请手动覆盖 1_Data\003_指数量价跟踪 下的同名文件（脚本不会自动搬运它）。
